# Internacionalización (i18n)

El sitio es bilingüe: español (default) e inglés.

## Configuración Astro

En `astro.config.mjs`:

```javascript
export default defineConfig({
  i18n: {
    defaultLocale: 'es',
    locales: ['es', 'en'],
    routing: {
      prefixDefaultLocale: true,
    },
  },
});
```

## Rutas

| Español | Inglés |
|---------|--------|
| `/es` | `/en` |
| `/es/blog` | `/en/blog` |
| `/` → redirect `/es` | — |

## Estrategia de contenido

### UI strings — `src/i18n/ui.ts`

Cadenas de interfaz centralizadas: navegación, labels de sección, CTAs, mensajes de estado.

```typescript
export const ui = {
  es: {
    'nav.projects': 'Proyectos',
    // ...
  },
  en: {
    'nav.projects': 'Projects',
    // ...
  },
} as const;
```

Helpers en `src/i18n/utils.ts`:

- `getLangFromUrl(url: URL): 'es' | 'en'`
- `useTranslations(lang: 'es' | 'en')` — retorna función `(key) => string`

### Contenido del portafolio — `src/data/portfolio.{lang}.ts`

Archivos separados por idioma con la misma estructura TypeScript (tipos en `src/types/portfolio.ts`).

### Blog (Fase 2) — carpetas separadas

```
src/content/blog/
├── es/
│   └── migrando-a-tanstack.md
└── en/
    └── migrating-to-tanstack.md
```

Cada artículo es independiente; no se traduce automáticamente.

## SEO — hreflang

El layout incluye alternates para cada página:

```html
<link rel="alternate" hreflang="es" href="https://example.com/es" />
<link rel="alternate" hreflang="en" href="https://example.com/en" />
```

## Selector de idioma

Componente en `Header.astro`: enlaces ES / EN que apuntan a la ruta equivalente en el otro idioma.

## Convenciones

- Código (variables, funciones, archivos): inglés
- Textos visibles al usuario: según locale activo
- Commits y documentación interna: español o inglés
