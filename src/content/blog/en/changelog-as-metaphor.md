A portfolio isn't a static resume: it's the history of what you've built.

## The metaphor

Each site section maps to a `CHANGELOG.md`:

| Section | Metaphor |
|---------|----------|
| Hero | Current release |
| Projects | `+ added` |
| Experience | Previous releases |
| Blog | Release notes |

Change types (`added`, `improved`, `fixed`, `deprecated`) give immediate context: is this new, an improvement, or a fix?

## Why not thermal receipt

The thermal receipt aesthetic is overused. Changelog, on the other hand, reflects a real habit of software writers: documenting changes, versioning decisions, and communicating impact.

## Export CV as release note

Phase 3 will carry this metaphor to PDF: a downloadable CV in release note format, not a generic Word table.
